## Shell 执行问题排查

exec 工具间歇性无法执行命令（无输出、无报错），排查过程：

### 现象
- `exec` 普通模式（非 PTY）执行命令完全无输出
- 子代理也超时失败
- `web_fetch` 和 `read`/`write` 正常

### 解决方法
使用 **PTY 模式** (`pty: true`) 执行 shell 命令，恢复正常：
```
exec(command, { pty: true, timeout: 90 })
```

### 经验总结
- exec 非PTY模式可能间歇性失灵，遇到无输出情况优先尝试 `pty: true`
- PTY 模式下 `yieldMs` 需要设够时间（网络操作如 git push 需要较长时间）
- git clone 超时时可用 `--depth 1` 浅克隆加速
- push 时 URL 内嵌 token：`https://user:token@github.com/user/repo.git`

## GitHub Token 安全
